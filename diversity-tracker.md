# Diversity Tracker

The Architect reads this file before selecting a genre profile and updates it after completing the Architect phase for each new novel. The purpose is to ensure that automatically generated books do not cluster around the same category, genre profile, voice, tone, POV, or setting.

Only future books in these approved categories may be generated automatically:
- Sci-Fi
- Fantasy
- Thriller
- Military SF / Space

Legacy entries outside those categories remain below for historical reference, but they are not approved templates for new work.

## Rules for the Architect

- Select only from the approved profiles in `genres.md`
- Do not repeat the same genre profile within the last 3 books when another approved profile would fit
- Do not repeat the same POV within the last 2 books
- Do not repeat the same tone descriptor within the last 2 books
- Do not repeat the same setting type within the last 3 books
- Maximize contrast with the most recent entries on every dimension possible without leaving the approved categories

## Dimensions Tracked

For each book, record:
- **category**: Sci-Fi / Fantasy / Thriller / Military SF / Space / Legacy
- **genre**: The exact genre profile selected from `genres.md`
- **pov**: First person / Third limited / Third omniscient / Multiple / Epistolary
- **tone**: The primary tone descriptor
- **setting_type**: Near-future / Deep space / Secondary-world fantasy / Contemporary / etc.
- **protagonist_type**: Brief description
- **sentence_rhythm**: Dominant rhythm
- **christian_integration**: How the worldview is integrated
- **structure**: Story structure used
