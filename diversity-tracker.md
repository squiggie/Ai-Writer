# Diversity Tracker

The Architect reads this file before selecting a genre profile and updates it after completing the Architect phase for each new novel. The purpose is to ensure that automatically generated books do not cluster around the same genre, voice, tone, POV, or setting.

## Rules for the Architect

- Do not repeat the same genre within the last 3 books
- Do not repeat the same POV within the last 2 books
- Do not repeat the same tone descriptor within the last 2 books
- Do not repeat the same setting type within the last 3 books
- Maximize contrast with the most recent entry on every dimension possible

## Dimensions Tracked

For each book, record:
- **genre**: The genre profile selected from genres.md
- **pov**: First person / Third limited / Third omniscient / Multiple / Epistolary
- **tone**: The primary tone descriptor (e.g. grave, comic, tense, luminous, unsettling)
- **setting_type**: Contemporary / Near-future / Historical / Fantasy / Rural / Urban / At sea / etc.
- **protagonist_type**: Brief description (e.g. male analyst, young woman, elderly priest, ensemble)
- **sentence_rhythm**: Dominant rhythm (e.g. spare declarative, lyrical long-form, clipped active)
- **christian_integration**: How the worldview is integrated (e.g. quiet grace, moral confrontation, providence as comedy, faith tested)
- **structure**: Story structure used (Seven-Point / Hero's Journey / Freytag / Story Circle / Fichtean / Non-Linear)
